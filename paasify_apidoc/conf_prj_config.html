Paasify Project settings


Configure main project settings. It provides global settings but also defaults vars and tags for all stacks.

  • Project configuration
  • Empty
root oneOf Project configuration
Type: object Default: {}

Configure project as a dict value. Most of these settings are overridable via environment vars.

 No Additional Properties
Example:

{
    "config": {
        "namespace": "my_ns1",
        "vars": [
            {
                "my_var1": "my_value1"
            }
        ],
        "tags": [
            "tag1",
            "tag2"
        ]
    }
}

root oneOf Project configuration vars

Environment configuration. Paasify leave two choices for the configuration, either use the native dict configuration or use the docker-compatible format

  • Env configuration as list
  • Env configuration as dict (Compat)
  • Unset
root oneOf Project configuration vars oneOf Env configuration as list
Type: array Default: []

Configure variables as a list. This is the recommended way asit preserves the variable parsing order, useful for templating. This format allow multiple configuration format.
Example:

{
    "env": [
        {
            "MYSQL_ADMIN_USER": "MyUser"
        },
        {
            "MYSQL_ADMIN_DB": "MyDB"
        },
        {
            "MYSQL_ENABLE_BACKUP": true
        },
        {
            "MYSQL_BACKUPS_NODES": 3
        },
        {
            "MYSQL_NODE_REPLICA": null
        },
        "MYSQL_WELCOME_STRING=Is alway a string"
    ]
}

Each additional property must conform to the following schema

root oneOf Project configuration vars oneOf Env configuration as list additionalProperties

Environment configuration. Paasify leave two choices for the configuration, either use the native dict configuration or use the docker-compatible format

  • Variable definition as key/value
  • Variable definition as string (Compat)
root oneOf Project configuration vars oneOf Env configuration as list additionalProperties oneOf Variable definition as key/value
Type: object

Simple key value variable declaration, under the form of: {KEY: VALUE}. This does preserve value type.
Example:

{
    "env": [
        {
            "MYSQL_ADMIN_USER": "MyUser"
        },
        {
            "MYSQL_ADMIN_DB": "MyDB"
        }
    ]
}

All properties whose name matches the following regular expression must respect the following conditions

Property name regular expression: ^[A-Za-z_][A-Za-z0-9_]*$
root oneOf Project configuration vars oneOf Env configuration as list additionalProperties oneOf Variable definition as key/value Environment Key value
Type: object

Value must be serializable type
root oneOf Project configuration vars oneOf Env configuration as list additionalProperties oneOf Variable definition as string (Compat)
Type: string

Value must be a string, under the form of: KEY=VALUE. This does not preserve value type.

Must match regular expression: ^[A-Za-z_][A-Za-z0-9_]*=.*$
Example:

{
    "env": [
        "MYSQL_ADMIN_USER=MyUser",
        "MYSQL_ADMIN_DB=MyDB"
    ]
}
root oneOf Project configuration vars oneOf Env configuration as dict (Compat)
Type: object Default: {}

Configure variables as a dict. This option is only proposed for compatibility reasons. It does not preserve the order of the variables.
Example:

{
    "env": {
        "MYSQL_ADMIN_USER": "MyUser",
        "MYSQL_ADMIN_DB": "MyDB",
        "MYSQL_ENABLE_BACKUP": true,
        "MYSQL_BACKUPS_NODES": 3,
        "MYSQL_NODE_REPLICA": null
    }
}

root oneOf Project configuration tags

Determine a list of tags to apply

  • List of tags
  • Unset
root oneOf Project configuration tags oneOf List of tags
Type: array Default: []

Define a list of tags. You can interact in few ways with tags. Tags can support boths syntaxes at the same time.
Example:

{
    "tags": [
        "my_tagg",
        "~my_prefix_tag",
        "my_collection:my_prefix_tag",
        {
            "other_tag": {
                "specific_conf": "val1"
            }
        },
        {
            "my_collection:another_tag": null
        },
        {
            "~ignore_this_tag": {
                "specific_conf": "val1"
            }
        }
    ]
}

Each additional property must conform to the following schema

root oneOf Project configuration tags oneOf List of tags StackTag configuration

Tag definition. It support two formats at the same time: as string or dict. If the name is prefixed with a !, then it is removed from the processing list (both vars, docker-file and jsonnet processing).

  • As string
  • As object
root oneOf Project configuration tags oneOf List of tags StackTag configuration oneOf As string
Default: ""

  • Tag name: $tag_name
